# 写作指南

欢迎来到 Astro Cactus 的写作部分！这个指南将帮助你创建高质量的博客内容。

## 📖 内容类型

Astro Cactus 支持两种主要内容类型：

### 📝 文章 (Posts)
位置：`src/content/post/`

- 完整、正式的长篇内容
- 支持标签、封面、更新日期等
- 用于正式发布的内容
- 出现在文章列表和 RSS 源中

👉 [文章写作指南](./posts.md)

### 📔 笔记 (Notes)
位置：`src/content/note/`

- 简洁的快速记录和学习笔记
- 更灵活的结构
- 用于知识积累
- 适合代码片段、参考资料等

👉 [笔记写作指南](./notes.md)

### 🏷️ 标签 (Tags)
位置：`src/content/tag/`（可选）

- 为特定标签创建自定义页面
- 提供标签的介绍和说明
- 引导读者浏览相关内容

👉 [标签管理指南](./tags.md)

## 🚀 快速开始

### 创建新文章

1. 在 `src/content/post/` 中创建 `.md` 文件
2. 添加 frontmatter（文章元数据）
3. 编写 Markdown 内容
4. 保存并访问 `localhost:3000/posts/filename`

```markdown
---
title: "我的第一篇文章"
description: "文章简介"
publishDate: "2024-04-29T10:00:00Z"
tags: ["astro", "web"]
---

# 开始写作吧！
```

### 创建新笔记

1. 在 `src/content/note/` 中创建 `.md` 文件
2. 添加标题和日期
3. 编写笔记内容
4. 保存并访问 `localhost:3000/notes/filename`

```markdown
---
title: "学习笔记"
publishDate: "2024-04-29T10:00:00Z"
---

## 核心概念

- 点 1
- 点 2
```

## 📋 内容清单

创建新内容时，使用这个清单确保质量：

### 文章清单

- [ ] 标题清晰且有吸引力（60 字符以内）
- [ ] 描述准确（50-160 字符）
- [ ] 发布日期正确
- [ ] 选择了 2-5 个相关标签
- [ ] 内容结构清晰，有合理的标题层级
- [ ] 使用了代码高亮（如有代码）
- [ ] 添加了有意义的封面（如有）
- [ ] 检查拼写和语法
- [ ] 测试所有链接

### 笔记清单

- [ ] 标题简洁明了
- [ ] 内容有清晰的结构
- [ ] 关键点突出
- [ ] 如有代码，格式正确
- [ ] 信息准确

## 💡 创意写作建议

### 文章主题灵感

- **教程** - 分享你学到的技能
- **案例分析** - 分析真实项目
- **工具评测** - 评比和对比
- **最佳实践** - 分享经验教训
- **问题解决** - 记录调试过程
- **行业新闻** - 讨论新技术

### 笔记主题灵感

- **学习笔记** - 学习新技术的记录
- **参考指南** - API 或工具速查表
- **代码片段** - 常用代码收集
- **阅读总结** - 书籍或文章摘录
- **项目文档** - 项目信息记录

## 🎨 格式化技巧

### 代码块

使用语言标识提供语法高亮：

````markdown
```javascript title="example.js" {1,3}
const x = 1;
const y = 2;
console.log(x + y);
```
````

### 强调

使用不同的强调方式：

```markdown
> [!NOTE]
> 重要信息

> [!WARNING]
> 警告信息

> [!TIP]
> 有用建议

> [!DANGER]
> 危险内容
```

### 列表和表格

```markdown
- 项目 1
- 项目 2
  - 子项目

| 列 1 | 列 2 |
|------|------|
| 数据 | 数据 |
```

## 🔍 SEO 优化建议

1. **标题** - 包含关键词，保持在 60 字以内
2. **描述** - 准确概括内容，50-160 字符
3. **标签** - 使用相关的、一致的标签
4. **链接** - 链接到相关文章和外部资源
5. **格式** - 使用合理的标题层级，便于搜索引擎理解

## 📤 发布前

3. 验证 Markdown 格式无误
4. 检查拼写和语法
5. 审视内容是否清晰有用
