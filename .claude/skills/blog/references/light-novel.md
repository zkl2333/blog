# 轻小说体指南

2026-06-12 在《AI 让我膨胀之我要画板》（sleepy-sensor-v1-bringup）上首发，用户连说「我太喜欢这种风格了」「再轻小说一点」；同日把《墨水屏时钟没居中》（eink-render-pure-node）也改了过去。从此是本博客的招牌体裁之一。

## 何时用（体裁跟着故事形状走）

**适合**：排障记、翻案记、连锁暴走（小起因滚成大重构）——有悬念、有反转、有"我和 Claude"两个角色可以演对手戏的素材。

**不适合**：思辨 / 盘点类（没有案子可破）、挽歌调子（会毁掉）。**不批量回写旧文**——用户拍板过：风格演化轨迹本身是存档的一部分，旧文留着当年轮；每篇都话数+伏笔，装置就从惊喜变套路（风格疲劳）。

## 标题

「XX 之 XX」式。用户自给的锚样本：**「AI 让我膨胀之我要画板」**——他原话「就很轻小说标题了」。正文开头可以 title drop（「某天宣布：我要画板。」）。

## 章节装置

话数体：`序章 → 第一话…第 N 话 → 幕间（过渡/呼吸段）→ 终章 → 尾声`。可选件：

- **卷末感想**——反思段的位置（"到底懂了没"）
- **附录**——架构图 / 设定资料的位置（mermaid 挂这儿，文库本设定资料集的味）
- **后记**——衍生事的位置（"后来单独开了个库"）
- **书挡**——「序章，完。」「第一卷，完。」首尾呼应；系列续作预告（"到时候，就是第二卷了"）

章节标题用完整短句带梗：「第一话：凶手就是你了，LDO」「第二话：病人压根没病」「第三话：这次是真的」「终章：丝印 1001」。

## 节拍

- **单句成段当吐槽 / 反应**：「叮咚。」「……为什么。」「天呐。」「绿了。」「能差三倍多。」——一篇里可以多处（这是体裁特权，覆盖 voice.md「独立短段最多 1 处」的默认值），但每个单句段必须有具体内容或真实情绪，空泛的"就这样。"仍然是病
- **剧本 vs 实际**：「按剧本，接下来应该是：……无事发生。／实际上是——／无限重启。」
- **立 flag**：「没什么契机，就是手痒。……——以上是 flag。」
- **连环动作快切**：「写构建脚本。配 workflow。推上去。盯着跑。第一轮，挂了。……绿了。」

## 对话体

Claude 升格为正经角色，直接引语「」独立成行：

> Claude 比我冷静：「先别猜，抓串口。」

用户自己的关键句也可引语化（「这很离谱。」「充电电阻应该是 1k 的吧？」）。转述他说过的话提炼成引语属于体裁惯例，但**语义必须是他真说过的**，措辞尽量贴原话。

## 主题装置（每篇换一套，别共用）

- **侦探卷宗**（sleepy-sensor 用过）：嫌疑人 / 调出档案（数据手册）/ 受害人 / 动机手法时间线 / 人赃并获当场结案 / 冤案 / 「侦探查到最后，发现凶手是自己」
- **连锁暴走**（eink-render-pure-node 用过）：连环否决 / 「那就自己编一个」+「反正动手的不是我」/ 公开处刑（CI 红叉）
- **伏笔明牌**：「至于那句『记得换回去』——请记住它，这是伏笔。」→ 回收时「对，就是那个伏笔。」
- 拟人化：板子「在反复喊：给我点活干」；医疗线（动手术 / 病人 / ICU / 出殡）

下一篇用新装置，这两套是占用状态。

## 底线（比体裁优先）

1. **事实、数字、时序、链接、代码块、mermaid、图和 alt 全部冻结**——只换讲法，不动事实
2. 内心戏和吐槽 OK，**编造具体事实细节不行**：发明的动作（"拿计算器啪啪一按"）、时长（"看了三秒"）、行为（"截图发朋友圈"）写之前自查，砍掉
3. **金句腔仍然是雷**：轻小说 ≠ 修辞堆叠。「这是 X，却不是 Y」对仗收束、三层工程化反讽，照样被用户点名"很 AI"（见 voice.md 被否构造表）。轻小说的爽点在节拍和角色，不在句子工整
4. 叙述者特权可以用（回忆者知道结局：「后来证明，这一跤摔的地方跟焊接半点关系没有」），但当下场景里的认知要符合当时（"案子当场就结了"是当时的笃定，翻案留给后文）
5. voice.md 的全部规则继续适用（破折号上限、口语校准、先决条件）

## 已发布文章改版流程

已发布（尤其置顶）的文章改体裁：**本地 commit、不 push**，把目录和关键段落贴给用户过目，他说「push」才上线。
